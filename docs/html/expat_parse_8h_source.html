109  /*
110  A handler for any characters in the document which wouldn't otherwise be handled. This includes both data
111  for which no handlers can be set (like some kinds of DTD declarations) and data which could be reported
112  but which currently has no handler set. The characters are passed exactly as they were present in the XML
113  document except that they will be encoded in UTF-8 or UTF-16. Line boundaries are not normalized. Note
114  that a byte order mark character is not passed to the default handler. There are no guarantees about how
115  characters are divided between calls to the default handler: for example, a comment might be split between
116  multiple calls. Setting the handler with this call has the side effect of turning off expansion of references
117  to internally defined general entities. Instead these references are passed to the default handler.

119  virtual void defaultHandler(const XML_Char *s, int len);

121  /*
122  Set an external entity reference handler. This handler is also called for processing an external DTD
123  subset if parameter entity parsing is in effect. (See XML_SetParamEntityParsing.)
124 
125  The context parameter specifies the parsing context in the format expected by the context argument to
126  XML_ExternalEntityParserCreate. code is valid only until the handler returns, so if the referenced entity
127  is to be parsed later, it must be copied. context is NULL only when the entity is a parameter entity,
128  which is how one can differentiate between general and parameter entities.
129 
130  The base parameter is the base to use for relative system identifiers. It is set by XML_SetBase and may
131  be NULL. The publicId parameter is the public id given in the entity declaration and may be NULL. systemId
132  is the system identifier specified in the entity declaration and is never NULL.
133 
134  There are a couple of ways in which this handler differs from others. First, this handler returns a status
135  indicator (an integer). XML_STATUS_OK should be returned for successful handling of the external entity
136  reference. Returning XML_STATUS_ERROR indicates failure, and causes the calling parser to return an
137  XML_ERROR_EXTERNAL_ENTITY_HANDLING error.
138 
139  Second, instead of having the user data as its first argument, it receives the parser that encountered
140  the entity reference. This, along with the context parameter, may be used as arguments to a call to
141  XML_ExternalEntityParserCreate. Using the returned parser, the body of the external entity can be
142  recursively parsed.
143 
144  Since this handler may be called recursively, it should not be saving information into global or static variables.
145  */

166  /*
167  A handler to deal with encodings other than the built in set.
168 
169  If the handler knows how to deal with an encoding with the given name, it should fill in the info data
170  structure and return XML_STATUS_OK. Otherwise it should return XML_STATUS_ERROR. The handler will be
171  called at most once per parsed (external) entity.
172 
173  The map array contains information for every possible possible leading byte in a byte sequence. If the
174  corresponding value is >= 0, then it's a single byte sequence and the byte encodes that Unicode value.
175  If the value is -1, then that byte is invalid as the initial byte in a sequence. If the value is -n,
176  where n is an integer > 1, then n is the number of bytes in the sequence and the actual conversion is
177  accomplished by a call to the function pointed at by convert. This function may return -1 if the sequence
178  itself is invalid. The convert pointer may be null if there are only single byte codes. The data parameter
179  passed to the convert function is the data pointer from XML_Encoding. The string s is NOT nul-terminated
180  and points at the sequence of bytes to be converted.
181 
182  The function pointed at by release is called by the parser when it is finished with the encoding. It may be NULL.
183 
184  typedef struct {

230  /*
231  A handler for element declarations in a DTD. The handler gets called with the name of the element
232  in the declaration and a pointer to a structure that contains the element model. It is the application's
233  responsibility to free this data structure using XML_FreeContentModel.
234 
235  The model argument is the root of a tree of XML_Content nodes. If type equals XML_CTYPE_EMPTY or
236  XML_CTYPE_ANY, then quant will be XML_CQUANT_NONE, and the other fields will be zero or NULL. If type is
237  XML_CTYPE_MIXED, then quant will be XML_CQUANT_NONE or XML_CQUANT_REP and numchildren will contain the
238  number of elements that are allowed to be mixed in and children points to an array of XML_Content structures
239  that will all have type XML_CTYPE_NAME with no quantification. Only the root node can be type XML_CTYPE_EMPTY,
240  XML_CTYPE_ANY, or XML_CTYPE_MIXED.
241 
242  For type XML_CTYPE_NAME, the name field points to the name and the numchildren and children fields will
243  be zero and NULL. The quant field will indicate any quantifiers placed on the name.
244 
245  Types XML_CTYPE_CHOICE and XML_CTYPE_SEQ indicate a choice or sequence respectively. The numchildren field
246  indicates how many nodes in the choice or sequence and children points to the nodes.

251  XML_CTYPE_MIXED,
252  XML_CTYPE_NAME,
